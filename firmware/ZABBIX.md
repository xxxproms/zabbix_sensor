# Как добавить Arduino-датчики температуры в Zabbix

Инструкция для **Zabbix 7.4.x**. Прошивка Arduino работает как **пассивный Zabbix-агент**: Zabbix сам подключается к Arduino по сети и запрашивает данные.

> Прошивка: [zabbix_ds18b20.ino](zabbix_ds18b20/zabbix_ds18b20.ino)
> Остальная документация: [Схемы](SCHEMA_I_SOEDINENIYA.md) | [BOM](OBSHIY_BOM.md) | [Настройка и отладка](NASTR_AYKA_I_OTLADKA.md)

---

## Что понадобится перед началом

- Arduino с прошивкой залита и подключена к сети (кабель в ENC28J60).
- IP-адрес Arduino известен (смотрите в **Serial Monitor** после включения).
- Доступ к веб-интерфейсу Zabbix (логин с правами на создание хостов).
- С сервера Zabbix (или прокси) должна быть сетевая доступность до Arduino на порту **10050**.

---

## Какие данные умеет отдавать Arduino

| Ключ (Key) | Что возвращает | Пример ответа |
|-------------|----------------|---------------|
| `agent.ping` | Жив ли агент | `1` |
| `env.temp` | Температура с датчика 1 | `23.50` |
| `env.temp1` | Температура с датчика 2 | `24.75` |

Если запросить несуществующий ключ, Arduino вернёт `ZBX_NOTSUPPORTED`.

---

## Пошаговая настройка в Zabbix

### Шаг 1. Проверьте связь с Arduino из консоли

Прежде чем настраивать в интерфейсе, убедитесь, что Zabbix-сервер (или прокси) вообще «видит» Arduino. Зайдите на сервер по SSH и выполните:

```bash
zabbix_get -s 10.10.0.50 -p 10050 -k agent.ping
```

Замените `10.10.0.50` на реальный IP вашей Arduino.

**Ожидаемый ответ:** `1`

Если ответ `1` — связь есть, переходите к шагу 2.

**Если ошибка / нет ответа:**
- Проверьте, что Arduino включена и в мониторе порта видно IP.
- Убедитесь, что кабель подключён к ENC28J60.
- Проверьте маршрутизацию (Arduino и сервер в одной подсети или есть маршрут).
- Убедитесь, что порт 10050 не заблокирован файрволом.

Также можно сразу запросить температуру:

```bash
zabbix_get -s 10.10.0.50 -p 10050 -k env.temp
```

**Ожидаемый ответ:** число вроде `23.50`.

---

### Шаг 2. Создайте группу хостов (необязательно, но удобно)

Группа нужна для порядка — чтобы Arduino-датчики не смешивались с серверами.

1. Откройте веб-интерфейс Zabbix.
2. Перейдите: **Сбор данных** → **Группы узлов сети** → кнопка **Создать группу узлов сети**.
   *(в английском интерфейсе: Data collection → Host groups → Create host group)*
3. Введите имя, например: `Датчики температуры`.
4. Нажмите **Добавить**.

Если не хотите создавать группу — можно использовать любую существующую.

---

### Шаг 3. Создайте хост (узел сети)

Хост в Zabbix — это ваша Arduino.

1. Перейдите: **Сбор данных** → **Узлы сети** → кнопка **Создать узел сети**.
   *(Data collection → Hosts → Create host)*

2. Заполните поля:

   | Поле | Что вписать |
   |------|-------------|
   | **Имя узла сети** | Любое понятное имя, например `arduino-sensor-01` |
   | **Группы** | Выберите группу из шага 2 (или любую другую) |

3. В блоке **Интерфейсы** нажмите **Добавить** → выберите **Агент**:

   | Поле | Что вписать |
   |------|-------------|
   | **IP-адрес** | IP вашей Arduino (например `10.10.0.50`) |
   | **DNS-имя** | Оставьте пустым |
   | **Подключение через** | Выберите **IP** |
   | **Порт** | `10050` |

4. Нажмите **Добавить** (внизу страницы) — хост создан.

На этом этапе хост будет «пустым» (без элементов данных) — это нормально.

---

### Шаг 4. Добавьте элемент данных «agent.ping» (проверка доступности)

Этот элемент покажет, отвечает ли Arduino.

1. Перейдите: **Сбор данных** → **Узлы сети** → нажмите на имя вашего хоста.
2. Перейдите на вкладку **Элементы данных**.
3. Нажмите **Создать элемент данных**.
4. Заполните:

   | Поле | Значение |
   |------|----------|
   | **Имя** | `Доступность агента` |
   | **Тип** | **Zabbix-агент** |
   | **Ключ** | `agent.ping` |
   | **Тип информации** | **Числовой (целое положительное)** |
   | **Интервал обновления** | `1m` |

5. Нажмите **Добавить**.

---

### Шаг 5. Добавьте элемент данных для первого датчика температуры

1. Снова нажмите **Создать элемент данных** (на той же вкладке).
2. Заполните:

   | Поле | Значение |
   |------|----------|
   | **Имя** | `Температура датчик 1` |
   | **Тип** | **Zabbix-агент** |
   | **Ключ** | `env.temp` |
   | **Тип информации** | **Числовой (с плавающей точкой)** |
   | **Единица измерения** | `°C` |
   | **Интервал обновления** | `1m` |

3. Нажмите **Добавить**.

---

### Шаг 6. Добавьте элемент данных для второго датчика температуры

1. Нажмите **Создать элемент данных**.
2. Заполните:

   | Поле | Значение |
   |------|----------|
   | **Имя** | `Температура датчик 2` |
   | **Тип** | **Zabbix-агент** |
   | **Ключ** | `env.temp1` |
   | **Тип информации** | **Числовой (с плавающей точкой)** |
   | **Единица измерения** | `°C` |
   | **Интервал обновления** | `1m` |

3. Нажмите **Добавить**.

---

### Шаг 7. Проверьте, что данные приходят

1. Подождите 1–2 минуты (Zabbix опрашивает агента с интервалом, который вы указали).
2. Перейдите: **Мониторинг** → **Последние данные**.
   *(Monitoring → Latest data)*
3. В фильтре выберите ваш хост.
4. Вы должны увидеть три элемента со значениями:
   - `Доступность агента` — значение `1`
   - `Температура датчик 1` — например `23.50`
   - `Температура датчик 2` — например `24.75`

**Если значения есть — всё работает!**

---

### Что делать, если данные не приходят

| Симптом | Что проверить |
|---------|--------------|
| Элемент в состоянии **Не поддерживается** | Ключ написан неверно. Проверьте: `env.temp` (не `env.temp0`, не `Env.Temp`). Регистр важен! |
| Нет данных, но элемент **поддерживается** | Подождите ещё 2–3 минуты, Zabbix мог не успеть опросить |
| Ошибка соединения | `zabbix_get` с сервера тоже не работает → проблема в сети (IP, кабель, файрвол) |
| Arduino перезагрузилась, IP сменился (DHCP) | Обновите IP в настройках хоста в Zabbix. Либо задайте статический IP в скетче |
| `DS18B20 count: 0` в мониторе порта | Датчики не найдены — проверьте пайку и резистор 4.7 кОм |

---

## Дополнительно: прокси

Если ваш Zabbix использует **прокси** для опроса:

1. В настройках хоста в поле **Наблюдается прокси** выберите нужный прокси.
2. Убедитесь, что именно этот прокси имеет сетевой доступ до Arduino.
3. Команду `zabbix_get` тоже запускайте **с прокси**, а не с сервера.

---

## Дополнительно: триггеры (оповещения)

Если хотите получать уведомление при перегреве:

1. Откройте хост → вкладка **Триггеры** → **Создать триггер**.
2. Имя: `Высокая температура датчик 1`.
3. Выражение: `last(/arduino-sensor-01/env.temp)>40`
   *(замените `arduino-sensor-01` на имя вашего хоста, `40` — на порог в градусах)*
4. Важность: **Высокая** (или по вкусу).
5. **Добавить**.

## Справочник: формат протокола Zabbix (для разработчиков)

> Этот раздел нужен только если вы модифицируете прошивку или отлаживаете проблемы на уровне пакетов.

Запрос от Zabbix к Arduino:

```
[ZBXD] [0x01] [8 байт длины payload, little-endian] [payload = ASCII ключ]
  4б     1б              8 байт                         N байт
```

Итого заголовок: **13 байт**. В прошивке это константа `ZBX_HEADER_LEN`.

Ответ от Arduino:

```
[ZBXD] [0x01] [8 байт длины ответа, little-endian] [ответ в ASCII]
```

Проверка дампом с сервера:

```bash
tcpdump -i any -nn -X tcp port 10050 and host 10.10.0.50
```
