# Zabbix: ключи агента и проверка протокола

Документ и прошивка согласованы с пассивным протоколом **Zabbix agent** на **Zabbix 7.4.8** (server/proxy опрашивает устройство на порту **10050**). Для других версий 7.x обычно достаточно тех же настроек элемента; при сбоях сверьте дамп TCP (ниже).

Прошивка: [zabbix_ds18b20/zabbix_ds18b20.ino](zabbix_ds18b20/zabbix_ds18b20.ino) (порт **10050**). Монтаж датчиков: [MONTAZH_I_PAYKA_DS18B20.md](MONTAZH_I_PAYKA_DS18B20.md).

## Ключи (Zabbix agent / passive checks)

| Ключ | Назначение |
|------|------------|
| `agent.ping` | Проверка доступности (ответ `1`) |
| `env.temp` | Температура, датчик 1 (ROM `tempSensor1`) |
| `env.temp1` | Датчик 2 |
| `env.temp2` | Датчик 3 |
| `env.temp3` | Датчик 4 |

Любой другой ключ → тело ответа `ZBX_NOTSUPPORTED` с корректным заголовком `ZBXD`.

## Пример элемента данных в Zabbix

1. **Создайте хост** с интерфейсом **Agent** → IP Arduino, порт `10050`.
2. Тип элемента: **Zabbix agent** (пассивный опрос сервером/proxy).
3. Ключ: например `env.temp`.
4. Тип информации: **Numeric (float)** или **Text** (прошивка отдаёт строку вида `23.50`).

Повторите элементы для `env.temp1` … `env.temp3`. Для доступности можно добавить элемент с ключом `agent.ping` и триггер/простую проверку.

## Формат кадра (сверка с вашим сервером)

Входящий запрос от Zabbix (пассивный агент):

- **13 байт заголовка**: `ZBXD` (4) + версия `0x01` (1) + **длина полезной нагрузки** как **uint64 little-endian** (8).
- Далее **ровно `payloadLen` байт** — ASCII ключа (часто с `\n` в конце; прошивка обрезает `\r`/`\n`).

В прошивке смещение начала ключа задано как **`ZBX_PAYLOAD_OFFSET`** (= 13). Итоговый размер кадра: `13 + payloadLen`.

Ответ:

- `ZBXD` + `0x01` + uint64 LE длины тела + **непрерывные байты** тела (без нулевого терминатора в потоке).

### Как проверить дампом

На машине с Zabbix или рядом с сетью:

```bash
tcpdump -i any -nn -X tcp port 10050 and host <IP_ARDUINO>
```

Убедитесь, что:

1. После `5a 42 58 44 01` восемь байт длины совпадают с длиной хвоста пакета минус 13.
2. Ключ в хвосте начинается с байта **смещения 13** от начала **полезной части TCP** (если один кадр в одном сегменте).

Если ваш сервер шлёт **другую длину заголовка** или смещение ключа не 13 (редкие прокси/обёртки), измените в скетче константы **`ZBX_HEADER_LEN`** и **`ZBX_PAYLOAD_OFFSET`** под фактический размер **до** начала ASCII ключа (сохранив разбор **uint64 LE** в байтах 5–12 относительно начала заголовка протокола Zabbix).

## Ориентиры по железу и стеку

- ENC28J60 + **UIPEthernet**: стабильное питание 3.3 В модуля; в скетче может быть задан статический IP под **10.10.0.0/23** (`255.255.254.0`) — уникальный адрес на каждое устройство в диапазоне **10.10.0.30–10.10.1.254**.
- Альтернатива с большим комьюнити: **W5500** / W5100 и штатная библиотека Ethernet, либо готовый агент **[zabbuino](https://github.com/zbx-sadman/zabbuino)** с множеством команд и тем же стилем протокола.
