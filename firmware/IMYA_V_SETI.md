# Имя устройства в сети (поиск «не только по IP»)

Кратко:

| Способ | Когда работает | Что даёт |
|--------|----------------|----------|
| **Имя в DHCP (опция 12)** | Питание по **DHCP** (`Ethernet.begin(mac)` без статики), прошивка/библиотека шлют hostname | В списке клиентов **роутера** часто видно имя; иногда роутер регистрирует **имя.lan** |
| **Статическая запись DNS / файл hosts** | Любой способ получения IP, есть доступ к DNS или к ПК | `ping sensor01` или полное FQDN по вашим правилам |
| **Имя хоста в Zabbix** | Всегда | Удобное имя в мониторинге — **не** заменяет DNS в ОС |

Прошивка по умолчанию использует **статический IP** (`USE_STATIC_IP`) — в этом режиме **DHCP hostname не отправляется**, роутер не узнает имя из запроса DHCP.

---

## 1. Имя в списке DHCP-клиентов (UIPEthernet)

В библиотеке **UIPEthernet** в файле `Dhcp.h` задано:

```cpp
#define HOST_NAME "ENC28J"
```

При запросе адреса по DHCP в пакет добавляется **hostname** (часто в интерфейсе роутера видно что-то вроде `ENC28J` + суффикс из MAC).

Чтобы задать **своё** короткое имя (латиница, без пробелов, например `zbx-temp-01`):

### Шаг A — однократная правка библиотеки

Найдите в папке библиотек Arduino файл  
`.../libraries/UIPEthernet/Dhcp.h`  
и замените строку `#define HOST_NAME "ENC28J"` на:

```cpp
#ifndef HOST_NAME
#define HOST_NAME "ENC28J"
#endif
```

(После обновления библиотеки через менеджер правку может потребоваться внести снова.)

### Шаг B — в скетче **до** `#include <UIPEthernet.h>`

В начале [zabbix_ds18b20.ino](zabbix_ds18b20/zabbix_ds18b20.ino) **выше** строки `#include <UIPEthernet.h>` добавьте, например:

```cpp
#define HOST_NAME "zbx-temp-01"
```

Ограничения: имя должно быть **коротким** (в коде DHCP длина связана с буфером; надёжно — до **~8–10 символов** латиницей и цифрами). Без кириллицы.

### Шаг C — включите DHCP в прошивке

Закомментируйте `#define USE_STATIC_IP` (или отключите статику), чтобы вызывался `Ethernet.begin(mac)` и устройство запрашивало адрес по DHCP. На роутере при необходимости задайте **резервирование** IP по MAC — тогда адрес стабильный, а имя в списке клиентов сохранится.

## 2. Статический IP (как у вас сейчас): как жить с «именем»

Пока включён **только статический** адрес, **hostname по DHCP не используется**.

Варианты:

1. **Роутер / DNS-сервер в организации** — вручную добавьте запись типа `A`: `sensor-rack-01.your.domain` → `10.10.0.50`.
2. **Файл hosts** на рабочем ПК (Windows `C:\Windows\System32\drivers\etc\hosts`, Linux/macOS `/etc/hosts`):  
   `10.10.0.50    sensor-rack-01`
3. **Zabbix** — в узле сети поле **Host name** / **Visible name** задаёт понятное имя в интерфейсе мониторинга; для доступа по имени из ОС это **не** подставляет DNS автоматически.

---

## 3. mDNS (`имя.local`) на Arduino Uno + ENC28J60

Реализация **Bonjour/mDNS** на AVR и стеке uIP возможна отдельными библиотеками, но это **много кода и RAM**, на Uno часто неудобно. Практичнее **имя на роутере/DNS** или **DHCP hostname** из п. 1.

---

## 4. Что выводить в Serial для себя

В скетче можно завести константу с **человекочитаемым именем** только для подписи в Serial (на монитор порта) — это не создаёт DNS, но помогает не путать платы при отладке.

См. необязательные строки в [zabbix_ds18b20.ino](zabbix_ds18b20/zabbix_ds18b20.ino) (`DEVICE_LABEL`).
